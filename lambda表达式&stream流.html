装逼程序员第一步：lambda表达式+stream流，不得不说还是肥肠好用的

lambda表达式

面向对象编程例如下面的代码是先创建匿名类对象，然后让对象做事情，但事实上，我们并不在意对象的哪个，只要他的函数（有点渣了bushi）

 Arrays.sort(nums,new Comparator<Integer>(){
     public int compare(Integer o1,Integer o2){
         return o1-o2;
     }
 });

于是出现函数式编程忽略面向对象复杂的语法，强调做什么而不是谁去做，lambda表达式是JDK 8开始后的一种语法方式

() -> {

}//示例如下
 Arrays.sort(nums,(Integer o1,Integer o2)->{
     return o1-o2;
 });
  • () 对应重写的方法的形参
  • -> 固定格式
  • {} 对应方法的方法体

那怎么知道lambda表达式重写了接口的哪个方法？注意点
lambda表达式只能简化函数式接口的匿名内部类的写法，函数式接口：有且只有一个抽象方法的接口叫做函数式接口，接口上方有@FunctionalInterface注解

//当一个接口有多个抽象方法，那么FunctionalInterface就会报错
@FunctionalInterface
interface function{
    public abstract void swim();
    public abstract void swim2();
}

lambda进一步省略
1、参数类型可以省略不写
2、如果只有一个参数，参数类型可以省略，同时（）也可以省略
3、如果方法体只有一行，大括号，分号，return可以省略不写，需要同时省略
省略核心：可推导出来的都可以省略

Arrays.sort(nums, (o1, o2) -> o1-o2);//最简写法

stream流

使用步骤
1、先得到一条stream流，将数据放上去
2、使用中间方法进行操作（过滤，转换）可以多次调用
3、终结方法（统计，打印）最后一步，调用完成后不能调用其他方法

获取stream流

单列集合获取stream流

 //单列集合获取stream流
 List<String> list = new ArrayList<>();
 //获取一条流水线，并将集合的数据放入
 Stream<String> stream = list.stream();
 //使用终结方法打印一下流水线的所有数据
 stream.forEach(new Consumer<String>() {
     @Override
     public void accept(String s) {
         System.out.println(s);
     }
 });
 //foreach是函数式接口，使用lambda简化就行
 list.stream().forEach(s-> System.out.println(s));

双列集合无法直接获取stream流，必须转为单列集合

 //双列集合stream流
 HashMap<String,String> map = new HashMap<>();
 //需要将双列转为单列
 map.keySet().stream().forEach(s-> System.out.println(s));
 map.entrySet().stream().forEach(s-> System.out.println(s.getValue()+s.getKey()));

数组需要借助Arrays工具类获取stream流

int arr[] = new int[10];
Arrays.stream(arr).forEach(num-> System.out.println(num));

中间方法


中间方法会返回新的stream流，原来的stream流只能使用一次，建议使用链式编程
修改stream流中的数据，不会影响原来集合或者数组中的数据
每个stream流对象只能被用一次，然后就会关闭了，之后就不能用了所以以后用stream流都用链式编程
下面这几个都巨简单，关键是map最难得好好学一学，也最常用

//filter过滤
list.stream().filter(s->s.startsWith("dyl")).forEach(s-> System.out.println(s));
//只操作前limit元素
list.stream().limit(3).forEach(s-> System.out.println(s));
//跳过前skip个元素
list.stream().skip(3).forEach(s-> System.out.println(s));
//distick 底层实现是hashSet，记得重写hashcode和equal方法
list.stream().distinct().forEach(s-> System.out.println(s));
//concat 只能合并两个流 这两个流尽量要类型一致，不一致会去共同父类，子类特殊方法就不能用了
Stream.concat(list.stream(),list.stream()).forEach(s-> System.out.println(s));

类型转换map详解（这个map可不是hashmap的键值对哈）

 ArrayList<String> arrayList = new ArrayList<>();
 Collections.addAll(arrayList,"dyl-15","dyt-19");
 //数据格式 ，姓名-年龄
 //需求：获取整形的年龄 string->int

 /**
  * Function接口第一个参数类型：流中原本的类型 第二个参数类型 返回值
  * apply 函数中 参数是原类型，返回值是新的类型
  * 在经过map处理后流内的数据就是新数据了
  */
 arrayList.stream().map(new Function<String, Integer>() {
     @Override
     public Integer apply(String s) {
         String[] split = s.split("-");
         return Integer.parseInt(split[1]);
     }
 }).forEach(age-> System.out.println(age));

lambda表达式写法

 arrayList.stream().map(s->{
     String[] split = s.split("-");
     return Integer.parseInt(split[1]);
 }).forEach(s-> System.out.println(s));

最终版简化代码如下,肥肠的nice

arrayList.stream().map(s->Integer.parseInt(s.split("-")[1])).forEach();

终结方法

因为他们的返回值不是stream对象了。所有不能再继续使用stream流了，所有是终结方法

count foreach没有什么好说的

long count = arrayList.stream().count();

toArray解释

 /**
  * new IntFunction<String[]>() 泛型是具体返回值类型的数组
  * apply形参 len是原来流中的个数
  * apply返回值：转换的数据具体类型
  * 方法体就是创建数组
  */
 String[] array = arrayList.stream().toArray(new IntFunction<String[]>() {
     @Override
     public String[] apply(int len) {
         return new String[len];
     }
 });

下面是lambda表达式的写法

String[] array1 = arrayList.stream().toArray(len -> new String[len]);

toMap解释，这是最难的我觉得,业务场景：将所有男生收集到map key为姓名 value为年龄，下面的版本使用的是toMap的双参版本，一个参数是键的生成规则，另外一个是值的生成规则，在这种情况下，键是一定不能重复的否则报错

 /**
  * toMap:参数一表示键的生成规则
  *       参数二表示值的生成规则
  *
  * 参数一：键的规则
  *      Function<String, String>()
  *      泛型一：原来流的数据类型
  *      泛型二：返回值的数据类型
  *      apply函数
  *      参数：流的数据
  *      方法体：得到键的逻辑
  *      返回值 键值
  *
  * 参数二：值的规则
  *     如上
  */
 Map<String, Integer> collect2 = arrayList.stream().filter(s -> "男".equals(s.split("-")[1]))
         .collect(Collectors.toMap(new Function<String, String>() {
             @Override
             public String apply(String s) {
                 return s.split("-")[0];
             }
         }, new Function<String, Integer>() {
             @Override
             public Integer apply(String s) {
                 return Integer.parseInt(s.split("-")[2]);
             }
         }));

lambda 表达式写法

 Map<String, Integer> collect3 = arrayList.stream().filter(s -> "男".equals(s.split("-")[1]))
         .collect(Collectors.toMap(key -> key.split("-")[0], 
                 value -> Integer.parseInt(value.split("-")[2])));

那么如果有一个场景，key可能重复的时候我们该怎么办，使用toMap的三参数规则这是真牛逼

 Map<String, Integer> collect4 = arrayList.stream().filter(s -> "男".equals(s.split("-")[1]))
         .collect(Collectors.toMap(new Function<String, String>() {
             @Override
             public String apply(String s) {
                 return s.split("-")[0];
             }
         }, new Function<String, Integer>() {
             @Override
             public Integer apply(String s) {
                 return Integer.parseInt(s.split("-")[2]);
             }
         }, new BinaryOperator<Integer>() {
             //将key重复时，value该如何操作
             @Override
             public Integer apply(Integer oldValue, Integer newValue) {
                 return newValue;
             }
         }));

转为lambda表达式

Map<String, Integer> collect5 = arrayList.stream().filter(s -> "男".equals(s.split("-")[1]))
        .collect(Collectors.toMap(key -> key.split("-")[0], 
                value -> Integer.parseInt(value.split("-")[2]),
                (oldValue,newValue)->newValue));
